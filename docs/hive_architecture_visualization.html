Current pipeline — logic per stage (humeo.pipeline.run_pipeline)

The row above is simplified: after pruning, the code runs keyframes + layout vision before render. Details below follow the real call order.

1 · Ingest — deterministic; cache skips re-download
Outputs

work_dir/source.mp4, transcript.json, optional source.info.json for manifest.

Logic
  • If ingest_complete(work_dir): reuse video + transcript (YouTube id cache path under HUMEO_CACHE_ROOT).
  • Else download via yt-dlp; extract audio; transcribe (OpenAI Whisper API or WhisperX per HUMEO_TRANSCRIBE_PROVIDER).
  • Transcript canonical JSON → transcript_sha256 — used by all downstream caches.
2 · Clip selection (Gemini, text-only)
Artifacts

clips.json, clips.meta.json, clip_selection_raw.json.

Algorithm
  • Prompts: clip_selection_system.jinja2 + clip_selection_user.jinja2 (override dir: HUMEO_PROMPTS_DIR).
  • Ask for a pool of candidates (default 12); each row is a Clip with virality_score ∈ [0,1].
  • Rank: sort by score; needs_review=True clips get a small penalty vs same score.
  • Keep set: all clips with score ≥ threshold (default 0.70) and not reviewed-out; if < min kept (5), take top N anyway up to max (8).
  • Response validated with Pydantic (humeo_core.schemas.Clip). Cache hit skips LLM when transcript hash + model match clips.meta.json.
2.25 · Hook detection (Gemini)
Artifacts

hooks.json, hooks.meta.json, hooks_raw.json.

Why

Clip selector often echoes the prompt’s [0, 3]s hook placeholder. That fingerprint would block start-trims in pruning. This stage overwrites hook_start_sec / hook_end_sec with a validated window per clip.

Logic
  • Validate: 0 ≤ hook_start < hook_end ≤ clip.duration, duration bounds (see hook prompt).
  • _looks_like_default_hook(0, 3) is treated as “no real hook” for later clamps if detection is off.
2.5 · Content pruning (coarse trim, Gemini)
Artifacts

prune.json, prune_raw.json, prune.meta.json.

Definition

Coarse = contiguous trim only: move in-point forward and out-point backward. No middle cuts. Writes trim_start_sec / trim_end_sec on each Clip.

Clamp algorithm (after LLM)
  • Cap total trim by prune_level (conservative / balanced / aggressive → max % of clip length).
  • Enforce MIN_CLIP_DURATION_SEC (50s) after trims.
  • If hook is real: do not trim start past hook_start − 0.25s nor end past tail that would remove hook tail (symmetric guard).
  • Optional: snap trims to transcript segment boundaries to avoid mid-sentence cuts.
  • Failure path: LLM/API error → no-op 0 / 0 trims for that clip; pipeline continues.
3 · Keyframes + layout vision (Gemini multimodal)
Artifacts

work_dir/keyframes/*.jpg, layout_vision.json, layout_vision.meta.json.

Logic
  • Build Scene per clip using clip_for_render(clip) — applies trims from Stage 2.5 so keyframe times match the export window.
  • extract_keyframes (ffmpeg snapshot) — one frame per clip window.
  • Vision: fixed JSON schema → LayoutInstruction per clip (layout enum + optional normalized bboxes for split modes). Prompt: inline GEMINI_LAYOUT_VISION_PROMPT in layout_vision.py.
  • Cache on transcript hash + clips hash + vision model id.
4 · Render (deterministic ffmpeg)
Outputs

output/short_<id>.mp4 + per-clip ASS under work_dir/subtitles/.

Logic
  • clip_for_render → effective_export_bounds: export window = [start,end] narrowed by trim_* only; hook fields cleared after mapping (hook was metadata for pruning).
  • humeo_core.primitives.compile: build ffmpeg graph from LayoutInstruction (five layouts, max two on-screen items); burn subtitles + title overlay.

Target map — what each block would add (paper vs repo)

A2 · Narrative context Backlogged

HIVE §3.1.5 style: one multimodal pass over sparse frames + transcript → narrative_context.json (global summary, per-scene captions, “value thread”). Feeds clip selection so charts/OCR matter before windows are chosen.

A3 · Character + dialogue fusion Backlogged

Paper path: diarization + face clusters + LLM reconciliation; OCR subtitles as anchor vs ASR. This repo uses transcript text only for selection; no speaker-ID or OCR-merge stage in the product pipeline yet.

A4 · Cross-episode memory Backlogged

HIVE memory module: persistent character + plot state across uploads. Here: single-video state only; artifacts are per work_dir.

B2 · Hook + outro boundaries In progress

Hook: implemented (Stage 2.25). Outro: no outro_* fields in schema; “ending” is prompt guidance inside clip selection + prune, not a separate timed boundary module like HIVE’s opening/ending Cartesian search.

B4 · Micro-pruning Backlogged

Would remove short silence spans and filler inside the clip (VAD / silencedetect / verbatim ASR) and splice in the compiler. Current compiler assumes one contiguous -ss/-t window per short.
